### Q6 討論：Context 管理方案

> 我的分析：你的核心需求是 **context 管理**，不是 timeline。時間軸其實是 metadata，
> 用來做事後分析（看 context switch 分佈）。所以我建議把 **context 管理做成主 UI**，
> 時間做成 **背景記錄 + 獨立分析面板**。
>
> 以下 3 個方案，請告訴我你偏好哪個，或想混搭哪些元素：

**方案 A：Context Dock（像瀏覽器 tab）**

最簡單直覺。上方是 context tab 列，主區域是當前 focus 的 tree。

```
┌─────────────────────────────────────────────────┐
│  [● Ctx A] [● Ctx B] [○ Ctx C]  [+New]         │ <- tab 列，● = active ○ = parked
├─────────────────────────────────────────────────┤
│                                                 │
│            ┌── feature-spec (md)                │
│  Auth系統 ─┤                                    │
│            ├── API設計 ── endpoints.md           │
│            └── DB schema (text)                 │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 優點：熟悉的 tab 模型、切換快（Cmd+1/2/3）、低學習成本
- 缺點：tab 多了會擠、看不到 context 狀態細節

---

**方案 B：Context Sidebar + Main Canvas（推薦）**

左側 sidebar 分層顯示 context 狀態，主區域全屏顯示 tree。

```
┌──────────────┬──────────────────────────────────┐
│  FOCUS       │                                  │
│  ► Auth系統  │          ┌── feature-spec (md)   │
│    (now)     │ Auth系統 ─┤                       │
│              │          ├── API設計              │
│  ACTIVE      │          │   └── endpoints.md    │
│  ● 前端重構  │          └── DB schema            │
│    (15m ago) │                                  │
│  ● CI/CD    │                                  │
│    (2h ago)  │                                  │
│──────────────│                                  │
│  PARKED      │                                  │
│  ○ 讀書筆記  │                                  │
│  ○ 週報素材  │                                  │
│──────────────│                                  │
│  [Insights]  │                                  │
└──────────────┴──────────────────────────────────┘
```

Context 生命週期：
```
  建立 ──► ACTIVE ──► PARKED ──► ARCHIVED
              │          │
              ◄──────────┘  (隨時喚回)
```

- `FOCUS`：當前正在看的那棵 tree（只有一個）
- `ACTIVE`：今天有碰過的 context，顯示「多久前」
- `PARKED`：手動暫停或系統偵測閒置 > N 分鐘後建議暫停
- `ARCHIVED`：完成/不再需要，隱藏但可搜尋

- 優點：一眼看到所有 context 狀態、自然的 active/parked 分層、sidebar 可收合不擋視線
- 缺點：比 tab 複雜一點點

---

**方案 C：Focus Mode + Quick Switcher（極簡）**

全屏只顯示 tree，零干擾。用快捷鍵叫出切換選單。

```
  正常模式（全屏 tree）：          切換模式（Cmd+K）：
┌───────────────────────┐    ┌───────────────────────┐
│                       │    │  ┌─────────────────┐  │
│          ┌── spec     │    │  │ > Auth系統  now  │  │
│ Auth系統 ─┤           │ => │  │   前端重構  15m  │  │
│          └── API設計  │    │  │   CI/CD    2h   │  │
│                       │    │  │   讀書筆記  park │  │
│                       │    │  └─────────────────┘  │
└───────────────────────┘    └───────────────────────┘
```

- 優點：最大化 focus、最少 UI 干擾、適合 ADHD
- 缺點：看不到其他 context 狀態（要叫出選單才看得到）

---

**時間/分析功能（獨立於上述三方案之外）：**

不論選哪個方案，時間都在背景自動記錄，獨立做成 Insights 面板：

**我的推薦：方案 B + 獨立 Insights**

理由：
1. Sidebar 讓你隨時知道「我手上有哪些 context」— 對 ADHD 很重要，減少遺忘焦慮
2. FOCUS/ACTIVE/PARKED 分層自然解決「context active 與否」的管理問題
3. 時間記錄靜默運作，不干擾工作，想看的時候打開 Insights
4. 鍵盤優先：Cmd+K quick switch、Cmd+P park current、Cmd+[ / ] 上下切換

### 方案 C 改良：Focus Mode + Status Bar + Quick Switcher

核心想法：加一條**極簡狀態列**在頂部，平常不干擾，但給滑鼠用戶一個入口，
同時把所有 context 管理集中到 **Quick Switcher** 裡。

**正常模式（全屏 tree + 頂部狀態列）：**

```
┌─────────────────────────────────────────────────┐
│ Auth系統          3 active ∙ 2 parked    [⌘K]  │ <- 狀態列（可點擊）
├─────────────────────────────────────────────────┤
│                                                 │
│              ┌── feature-spec (md)              │
│   Auth系統 ──┤                                  │
│              ├── API設計 ── endpoints.md         │
│              └── DB schema (text)               │
│                                                 │
│                                                 │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 狀態列只有一行，顯示：當前 context 名稱 + active/parked 數量 + 快捷鍵提示
- 點擊狀態列任何位置 = 打開 Quick Switcher（等同 Cmd+K）
- 不 hover 時幾乎不注意到它的存在

**Quick Switcher（Cmd+K 或點擊狀態列）：**

```
┌─────────────────────────────────────────────────┐
│ ░░░░░░░░░ (背景模糊：當前 tree) ░░░░░░░░░░░░░░ │
│                                                 │
│    ┌───────────────────────────────────────┐    │
│    │  Search context...              ⌘K   │    │
│    ├───────────────────────────────────────┤    │
│    │  FOCUS                               │    │
│    │  ► Auth系統                    now    │    │
│    ├───────────────────────────────────────┤    │
│    │  ACTIVE                              │    │
│    │  ● 前端重構                   15m    │    │
│    │  ● CI/CD                     2h     │    │
│    ├───────────────────────────────────────┤    │
│    │  PARKED                              │    │
│    │  ○ 讀書筆記              yesterday   │    │
│    │  ○ 週報素材                   2d     │    │
│    ├───────────────────────────────────────┤    │
│    │  [+ New]           [Insights]        │    │
│    └───────────────────────────────────────┘    │
│                                                 │
└─────────────────────────────────────────────────┘
```

**Quick Switcher 裡的操作（鍵盤 + 滑鼠都支援）：**

| 操作           | 鍵盤              | 滑鼠               |
|---------------|-------------------|---------------------|
| 切換 context  | ↑↓ 選取 → Enter   | 點擊 context        |
| Park context  | 選取 → P          | 右鍵 → Park         |
| Archive       | 選取 → A          | 右鍵 → Archive      |
| 刪除          | 選取 → D (確認)    | 右鍵 → Delete       |
| 新建          | N                 | 點 [+ New]          |
| 搜尋          | 直接打字           | 點搜尋欄            |
| 關閉          | Esc               | 點背景模糊區域       |

**Context 狀態自動管理：**

```
                    用戶切換到其他 context
  FOCUS ─────────────────────────────────► ACTIVE
    ▲                                        │
    │ 用戶選取切回                             │ 閒置超過 N 分鐘
    │                                        ▼
    └──────────────────────────────────── PARKED
                                             │
                                             │ 用戶手動 archive
                                             ▼
                                         ARCHIVED (隱藏，可搜尋)
```

- 切換 context 時，舊的自動變 ACTIVE
- 閒置超過設定時間 → 底部 toast：「CI/CD 閒置 2h，要 park 嗎？(Y/N)」
- 不強制，用戶可以忽略 toast，context 保持 ACTIVE

**你覺得這個改良版如何？還有想調整的地方嗎？**

**你的回答：**


針對方案 c 這裡 node 透過 icon 來讓 node 能夠被辨別是 pure text or file(image markdown)，針對檔案要有不同處理：
- markdown 為點擊編輯，而該 node title 跟檔案 title(不一樣)
- image 為直接渲染在畫面上
- 其他就連結到檔案就好

針對 context 管理，我覺得不要有閒置就 PARKED 應該說不要有這狀態，應該分 active/archived 然後如果近期沒有操作該 context 那就顯示灰色，這裡的近期是近一個小時（可調整）
