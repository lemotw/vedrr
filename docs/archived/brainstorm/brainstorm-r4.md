## Node 類型與顯示

每個 node 用 icon 區分類型，不同類型有不同互動方式：

```
  Auth系統 ──┬── [T] feature spec          <- 純文字 node，點擊 inline 編輯
             ├── [M] API設計               <- Markdown 檔案，點擊開編輯器
             │       (api-design.md)       <- 檔名顯示在 node title 下方（灰色小字）
             ├── [I] 架構圖                <- Image，直接渲染縮圖
             │   ┌─────────────┐
             │   │  (渲染圖片)  │
             │   └─────────────┘
             └── [F] schema.sql            <- 其他檔案，點擊用系統預設程式開啟
```

| Icon | 類型       | 互動                   |
| ---- | -------- | -------------------- |
| [T]  | 純文字      | 點擊 → inline 編輯       |
| [M]  | Markdown | 點擊 → 開內建編輯器，title 獨立 |
| [I]  | Image    | 直接渲染縮圖在 tree 上       |
| [F]  | 其他檔案     | 點擊 → 系統預設程式開啟        |

### 三層 Context 命名方案

幫你想了幾個「封存」的命名，配合前兩層的語意：

| 方案 | ACTIVE | 手動歸檔 | 自動封存（1天後） | 風格 |
|------|--------|---------|-----------------|------|
| A    | Active | Archived | **Vault** (封存庫) | 簡潔，像保險箱鎖起來 |
| B    | Active | Archived | **Sealed** (封存) | 直譯你說的封存，密封感 |
| C    | Active | Archived | **Cold** (冷藏) | 類似 AWS 冷存儲概念，技術味 |

**我推薦 A: Vault**

理由：短、好記、跟 Active/Archived 三個詞視覺長度接近，而且「vault」有「安全保管但不常取用」的語意，很符合封存的意思。


### 分離設計：Quick Switcher vs Context Manager

**Quick Switcher（⌘K）— 只做快速切換，保持精簡：**

```
    ┌───────────────────────────────────────┐
    │  Search active context...       ⌘K   │
    ├───────────────────────────────────────┤
    │  ► Auth系統                    now   │
    │  ● 前端重構                   15m   │
    │  ● CI/CD                     2h    │  <- 灰色
    ├───────────────────────────────────────┤
    │  [+ New]        [Context Manager]    │
    └───────────────────────────────────────┘
```

- 只顯示 active contexts
- 職責單一：快速切換 + 新建
- 底部有入口進 Context Manager

**Context Manager（獨立頁面/面板）— 管理所有 context 生命週期：**

**Context Manager — 大 pool，不分區，靠 filter/tag 管理：**

這裡我幫你想了幾種 tag/屬性方案：

**方案 1：狀態 tag + 自訂 tag**

```
┌──────────────────────────────────────────────────────────┐
│  Context Manager                                [Close] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Search...    Filter: [All] [#work] [#study] [#side]     │
│                                                          │
│  ┌───────────────────────────────────────────────────┐   │
│  │ ► Auth系統        active   #work       now        │   │
│  │ ● 前端重構        active   #work       15m        │   │
│  │ ● CI/CD          active   #work       2h         │   │
│  │ ○ 讀書筆記        archived #study      3d         │   │
│  │ ○ 週報素材        archived #work       5d         │   │
│  │ ◇ Q4 規劃         vault   #work       2w         │   │
│  │ ◇ 舊專案筆記       vault   #side       1mo        │   │
│  │ ...                                               │   │
│  └───────────────────────────────────────────────────┘   │
│                                                          │
│  Sort: [Recent] [Name] [Nodes]   Insights: 4 switches   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- 狀態（active/archived/vault）是自動欄位，不是 tag
- 用戶自訂 tag（#work, #study, #side...）用來分類
- Filter bar 可組合：點 #work = 只看 work 相關的 context
- 排序：按最近操作 / 名稱 / node 數量

**方案 2：無 tag，只靠搜尋 + 排序**

```
┌──────────────────────────────────────────────────────────┐
│  Context Manager                                [Close] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Search...    Sort: [Recent ▾]    Show: [All ▾]          │
│                                                          │
│  ┌───────────────────────────────────────────────────┐   │
│  │ ► Auth系統        ● active     5 nodes     now    │   │
│  │ ● 前端重構        ● active     3 nodes     15m    │   │
│  │ ● CI/CD          ● active     8 nodes     2h     │   │
│  │ ○ 讀書筆記        ○ archived   12 nodes    3d     │   │
│  │ ◇ Q4 規劃         ◇ vault     15 nodes    2w     │   │
│  │ ...                                               │   │
│  └───────────────────────────────────────────────────┘   │
│                                                          │
│  Show 下拉: All / Active only / Archived only / Vault    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- 最簡單，靠 Show 下拉篩選狀態、搜尋找特定 context
- 不用維護 tag，但隨著 context 數量增加可能比較難找

**方案 3（推薦）：自訂 tag + 狀態自動標記，但 UI 呈現為 flat list**

結合方案 1 的 tag 能力，但 pool 是 flat 的（不分區塊），用戶需要時才用 filter：

```
┌──────────────────────────────────────────────────────────┐
│  Context Manager                                [Close] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Search...                                               │
│  Tags: [All] [#work] [#study]    Status: [All ▾]         │
│                                                          │
│  ► Auth系統        ● now     #work           5 nodes    │
│  ● 前端重構        ● 15m    #work           3 nodes    │
│  ● CI/CD          ● 2h     #work           8 nodes    │
│  ○ 讀書筆記        ○ 3d     #study          12 nodes   │
│  ○ 週報素材        ○ 5d     #work           4 nodes    │
│  ◇ Q4 規劃         ◇ 2w     #work           15 nodes   │
│  ◇ 舊專案筆記       ◇ 1mo                    8 nodes    │
│                                                          │
│  Right-click: Activate / Archive / Delete / Add tag      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- Flat list 預設按最近操作排序
- 狀態用 icon 區分（● ○ ◇），一眼掃到
- Tag 是可選的，不加也不影響使用
- 需要的時候用 tag filter 或 status filter 縮小範圍
